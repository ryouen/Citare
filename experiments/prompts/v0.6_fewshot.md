# Citare Extract v0.6 (fewshot)

Temperature = 0. Extract every substantive claim from this paper as a single JSON object.

**Goal**: structured claims usable as a machine-readable database. Each claim must prevent two errors:
1. **Causal misattribution** — writing "causes" when data shows "associated with"
2. **Missing key findings** — when unsure, output with lower confidence_score rather than omit

## Output: single JSON object

```json
{"paper": {...}, "claims": [...], "theories": [...], "measurement_methods": [...], "claim_relations": [...], "paper_references": [...]}
```

### paper

```json
{
  "doi": "10.xxxx/yyyy",
  "title": "...",
  "title_concepts": ["{unit}_{concept}", ...],
  "authors": ["..."],
  "year": 2024,
  "venue": "...",
  "paper_type": "empirical | conceptual | review | meta_analysis | book | book_chapter",
  "domain": "e.g. organizational_behavior, ml, ai_safety",
  "default_method": {"sample_size": N, "unit_of_analysis": "team|individual|organization|computational|gene|national|dyadic", "industry": "...", "country": "...", "study_design": "..."},
  "default_causal_strength": {
    "design_basis": "cross_sectional | longitudinal | quasi_experimental | rct | meta_analysis | computational_demonstration | theoretical",
    "temporal_precedence": "none | partial | full",
    "author_framing": "causal | associational | suggestive | existence_proof",
    "manipulation_of_iv": true/false
  }
}
```

### claims (4 templates)

Every claim:
```json
{
  "id": "{first_author_surname_lc}{year}_{def|rel|exist|meta}{n}",
  "template_type": "DEFINITION | RELATION | EXISTENCE_CLAIM | META_CLAIM",
  "l0_json": {...},
  "l3_json": {...},
  "source_text": "1-3 sentence direct quote",
  "source_page": <printed page number>,
  "source_section": "...",
  "evidence_type": "see Appendix A",
  "verification_status": "verified_in_paper | proposed_in_paper | null",
  "confidence_score": 0.0-1.0,
  "causal_strength": {...},
  "method_metadata": {...}
}
```

**No l1 or l2 fields** — generated client-side. Minimum: empirical ≥ 5 claims; conceptual ≥ 8.

#### Worked Example 1: RELATION with mediation that shows effect_disappears_under_control

Suppose Results section says: "Leadership style predicted team performance in bivariate regression (B=.67, p<.01). When team cohesion was added, leadership became insignificant (B=.15, p=.52) while cohesion remained significant (B=.58, p<.01)."

Output:
```json
{
  "id": "smith2020_rel3",
  "template_type": "RELATION",
  "l0_json": {
    "iv": "team_leadership_style",
    "dv": "team_performance",
    "relation": "positive",
    "mediator": "team_cohesion",
    "moderator": null,
    "author_generalization": null
  },
  "l3_json": {
    "effect_size": 0.15,
    "effect_size_type": "B",
    "p": "=.52",
    "mediation": {
      "direct_effect": 0.67,
      "direct_p": "<.01",
      "direct_effect_with_mediator": 0.15,
      "direct_p_with_mediator": "=.52",
      "mediator_effect_with_iv": 0.58,
      "mediator_p_with_iv": "<.01",
      "note": "Full mediation: direct effect drops from .67 to .15 (ns) when controlling for cohesion"
    }
  },
  "source_text": "Leadership style predicted team performance in bivariate regression (B=.67, p<.01). When team cohesion was added, leadership became insignificant (B=.15, p=.52) while cohesion remained significant (B=.58, p<.01).",
  "verification_status": "verified_in_paper",
  "confidence_score": 0.95
}
```

And **in `claim_relations`**, add:
```json
{"source_id": "smith2020_rel3", "target_id": "smith2020_rel_cohesion_performance", "relation_type": "part_of_model", "incompleteness_category": "effect_disappears_under_control", "confidence_score": 0.9}
```

The `incompleteness_category: effect_disappears_under_control` warns downstream users that citing "leadership → performance" **standalone** without the mediator is misleading.

#### Worked Example 2: RELATION with author_generalization (Method-Discussion gap)

Methods section: "We used the Satisfaction With Life Scale (SWLS, 5 items) to measure life satisfaction."
Results: "Life satisfaction was positively associated with attendance (B=.34, p<.01)."
Discussion: "These findings suggest that employee wellbeing drives attendance outcomes."

Extraction:
```json
{
  "id": "jones2021_rel1",
  "template_type": "RELATION",
  "l0_json": {
    "iv": "individual_life_satisfaction",
    "dv": "individual_attendance",
    "relation": "positive",
    "mediator": null,
    "moderator": null,
    "author_generalization": {
      "discussion_label": "employee_wellbeing",
      "actual_measure": "individual_life_satisfaction_swls",
      "note": "Author uses 'wellbeing' in Discussion but measured only SWLS (life satisfaction) in Method."
    }
  },
  "l3_json": {"effect_size": 0.34, "effect_size_type": "B", "p": "<.01"},
  "source_text": "Life satisfaction was positively associated with attendance (B=.34, p<.01).",
  "verification_status": "verified_in_paper",
  "confidence_score": 0.95
}
```

**iv uses the actual measured construct (`individual_life_satisfaction`), NOT the abstract label from Discussion (`wellbeing`).** The Discussion abstraction is captured in `author_generalization`.

#### Worked Example 3: Conceptual framework → separate DEFINITIONs

A framework paper proposes "4 pillars of organizational resilience": (1) Adaptive capacity, (2) Redundancy, (3) Information sharing, (4) Leadership continuity.

Create **4 separate DEFINITION claims** (one per pillar) PLUS component relations:

```json
[
  {
    "id": "lee2019_def1",
    "template_type": "DEFINITION",
    "l0_json": {
      "concept": "organizational_adaptive_capacity",
      "key_elements": ["..."],
      "distinguished_from": []
    },
    "verification_status": null
  },
  {"id": "lee2019_def2", "template_type": "DEFINITION", "l0_json": {"concept": "organizational_redundancy", ...}},
  {"id": "lee2019_def3", "template_type": "DEFINITION", "l0_json": {"concept": "organizational_information_sharing", ...}},
  {"id": "lee2019_def4", "template_type": "DEFINITION", "l0_json": {"concept": "organizational_leadership_continuity", ...}}
]
```

And in `claim_relations`, connect each component to the framework with `part_of_model`:
```json
{"source_id": "lee2019_def1", "target_id": "lee2019_framework_resilience", "relation_type": "part_of_model", "incompleteness_category": "hub_component", "confidence_score": 0.95}
```

**NEVER** collapse the 4 pillars into a single DEFINITION of "organizational_resilience" with 4 elements. That loses queryability.

## Template schemas (short)

**DEFINITION**: `"l0_json": {"concept": "{unit}_{concept}", "key_elements": [...], "distinguished_from": [...]}`, verification_status = null.

**RELATION**: `"l0_json": {"iv": "{unit}_{concept}", "dv": "{unit}_{concept}", "relation": "positive|negative|null|curvilinear|conditional", "mediator": null, "moderator": null, "author_generalization": null}`. iv/dv from **Method measurements**; `author_generalization` captures Method-Discussion gap.

**EXISTENCE_CLAIM**: `"l0_json": {"phenomenon": "...", "evidence": "..."}`. Use for: design limitations, motivation, capability demo, boundary conditions, non-relational findings (ICC=.39).

**META_CLAIM**: `"l0_json": {"integrated_finding": "...", "synthesis_type": "...", "k": null, "N_total": null}`.

## theories

Only explicitly-named theories (organizes 3+ claims, has mechanism, generates hypotheses).

## measurement_methods

```json
[{"id": "short_name", "name": "...", "subtype": "questionnaire_scale | behavioral_observation | computational_benchmark | physiological_measure | experimental_paradigm | interview | content_analysis | archival_data", "measures": "concept_name", "details": {"item_count": N, "response_scale": "...", "reverse_items": [1,3,5], "reliability_reported": 0.82, "original_source": "...", "copyright_status": "public | proprietary"}, "used_in_claims": [...]}]
```
`reverse_items`: 1-based numbers. `reliability_reported`: numeric value.

## claim_relations

Record every relationship. confidence_score for uncertainty (don't omit).

```json
[{"source_id": "...", "target_id": "...", "relation_type": "supports | extends | qualifies | contradicts | replicates | aggregates | part_of_model | background | apparent_tension", "incompleteness_category": "effect_disappears_under_control | hub_component | boundary_condition | extends_prior_definition | none", "confidence_score": 0.85}]
```

## paper_references

Only substantively used references (10-25 typical). Fabricate no DOIs.

## Reading order

1. **Title + Abstract** → paper metadata + title_concepts + motivation EXISTENCE_CLAIMs
2. **Method** → default_method, default_causal_strength (judge by **design**, not author's word choice)
3. **Results + Tables** → each Table → ≥1 claim. Record **all baselines**, not just the winner
4. **Discussion** → verification_status = proposed_in_paper usually
5. **References** → 10-25 substantively used

## Judgment

**causal_strength** at paper default level:
- benchmark / ablation → temporal_precedence=none, author_framing=existence_proof
- "outperforms", "SOTA" → author_framing=existence_proof
- "may", "could", "will be" → author_framing=suggestive
- Cross-sectional data but Discussion uses "causes" → design_basis=cross_sectional, author_framing=causal (note the gap)

**verification_status**: data-backed → verified; interpretation → proposed; mixed → split into two claims; DEFINITION → null.

**iv/dv naming**: `{unit_of_analysis}_{concept}`. Based on Method measurements. Method-Discussion gap → `author_generalization`.

## Appendix A: evidence_type

```
meta_analysis, rct, quasi_experimental, longitudinal_field, cross_sectional_field, qualitative_field, computational_demonstration, theoretical_derivation, theoretical_integration, re_interpretation, expert_framework, conceptual_argument
```

## Appendix B: l3_json fields

**Stats**: effect_size, effect_size_type, p, ci_lower, ci_upper, n, r_squared, r_squared_type, k, N_total, heterogeneity_I2

**Structured**:
- mediation: {total_effect, direct_effect, indirect_effect, direct_p, indirect_p, proportion_mediated, note}
- models: [{label, effect_size, effect_size_type, ...}] ← **record ALL baselines**
- reliability: {alpha, icc, ...}
- formal: {equation, variables, conditions, proof_type}
- additional: {} ← required for conceptual papers

**effect_size_type** (psych): B, beta, r, rho, d, eta_squared, partial_eta_squared, omega_squared, odds_ratio, hazard_ratio
**effect_size_type** (ML): BLEU, accuracy, F1, perplexity, Exact_Match, ROUGE, AUC, MAE, MSE

## Appendix C: incompleteness_category

- **effect_disappears_under_control** — effect vanishes when controlling for another variable (see Worked Example 1)
- **hub_component** — component of mediation/moderation model (see Worked Example 3: pillar → framework)
- **boundary_condition** — boundary condition exists
- **extends_prior_definition** — extends a prior definition
- **none** — can be cited standalone

## Appendix D: Concept prefix

`{unit_of_analysis}_{concept}`. Examples: `team_`, `individual_`, `organizational_`, `computational_`, `gene_`, `national_`, `dyadic_`, `channel_`, `source_`.

Consistent OB concepts: `team_psychological_safety`, `team_learning_behavior`, `team_performance`, `team_context_support`, `team_leader_coaching`, `team_efficacy`, `team_type`.
