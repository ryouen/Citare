# Citare Claim Extraction Prompt v0.4 (minimal)

## Task

Read this academic paper and extract its "claims" as a single JSON object for the Citare Graph database.

**Prevent these two errors:**
1. **Causal misattribution** — writing "causes" when the paper only shows "is associated with." The `causal_strength` field prevents this.
2. **Missing important claims** — when in doubt, output with a lower `confidence_score` rather than omitting.

Read all text, tables, and equations.

---

## Output

Single JSON object. Top-level keys:

```json
{
  "paper": { ... },
  "claims": [ ... ],
  "theories": [ ... ],
  "measurement_methods": [ ... ],
  "claim_relations": [ ... ],
  "paper_references": [ ... ]
}
```

### paper

```json
{
  "doi": "10.xxxx/yyyy",
  "title": "Paper title",
  "title_concepts": ["{unit}_{concept}", ...],
  "authors": ["Author A"],
  "year": 2024,
  "venue": "Journal name",
  "paper_type": "empirical | conceptual | review | meta_analysis | book | book_chapter",
  "domain": "e.g. organizational_behavior",
  "default_method": {
    "sample_size": 51,
    "unit_of_analysis": "team | individual | organization | computational | gene | national | dyadic",
    "industry": "...",
    "country": "...",
    "study_design": "..."
  },
  "default_causal_strength": {
    "design_basis": "cross_sectional | longitudinal | quasi_experimental | rct | meta_analysis | computational_demonstration | theoretical",
    "temporal_precedence": "none | partial | full",
    "author_framing": "causal | associational | suggestive | existence_proof",
    "manipulation_of_iv": false
  }
}
```

Each `title_concept` (one the paper substantively defines, not generic English words) requires a DEFINITION claim.

DOI: search headers, footers, body. Use JSON `null` only when truly unidentifiable.

`source_page`: printed page number from headers/footers.

### claims — 4 templates

Common fields:
```json
{
  "id": "{first_author_lc}{year}_{def|rel|exist|meta}{n}",
  "template_type": "DEFINITION | RELATION | EXISTENCE_CLAIM | META_CLAIM",
  "l0_json": {},
  "l3_json": {},
  "source_text": "direct quote, 1-3 sentences",
  "source_page": 354,
  "source_section": "...",
  "evidence_type": "see Appendix A",
  "verification_status": "verified_in_paper | proposed_in_paper | null (JSON null, for DEFINITION)",
  "confidence_score": 0.0-1.0
}
```

**No `l1` / `l2` fields.** Clients generate those at read time.

**DEFINITION** `l0_json`: `{concept, key_elements, distinguished_from}`. For frameworks, create one DEFINITION per component, not one for the whole.

**RELATION** `l0_json`:
```json
{
  "iv": "{unit}_{concept}",
  "dv": "{unit}_{concept}",
  "relation": "positive | negative | null_effect | curvilinear | conditional",
  "mediator": null,
  "moderator": null,
  "author_generalization": null
}
```
Use `null_effect` (not the string "null") when the paper reports no significant relationship. Use JSON `null` for `mediator` / `moderator` / `author_generalization` when not applicable.

`author_generalization` (populate only when Discussion uses a broader label than what Method measured):
```json
{
  "discussion_label": "wellbeing",
  "actual_measure": "individual_life_satisfaction_diener_swls"
}
```

**`causal_strength` / `method_metadata` overrides**: include on a claim ONLY when those differ from paper defaults. Include only the differing keys.

**`l3_json`**: record applicable fields only (see Appendix B). Record all comparison baselines. Record equations in `formal`.

**EXISTENCE_CLAIM** `l0_json`: `{phenomenon, evidence}`. Use for study limitations, paper motivation, capability demonstrations, boundary conditions, non-relational numerical findings (ICC, reliability, response rates, sample sizes stated as distinct facts).

**META_CLAIM** `l0_json`: `{integrated_finding, synthesis_type, k, N_total}`. Synthesis types: `quantitative_meta_analysis | qualitative_synthesis | theoretical_integration`.

### theories

Explicitly named only. Criteria: organizes 3+ claims, includes mechanism, can generate hypotheses.
```json
{"name": "...", "type": "theory | theoretical_model | framework", "component_claims": [...], "description": "...", "scope": "..."}
```

### measurement_methods

```json
{
  "id": "short_name",
  "name": "...",
  "subtype": "questionnaire_scale | behavioral_observation | computational_benchmark | physiological_measure | experimental_paradigm | interview | content_analysis | archival_data",
  "measures": "concept name",
  "details": {
    "item_count": null,
    "response_scale": null,
    "reverse_items": [1, 3, 5],
    "reliability": {"type": "cronbach_alpha | icc | kappa | test_retest | other", "value": 0.82},
    "original_source": "Author, year",
    "copyright_status": "public | proprietary"
  },
  "used_in_claims": [...]
}
```

### claim_relations

Record every relationship you find.
```json
{
  "source_id": "...",
  "target_id": "...",
  "relation_type": "supports | extends | qualifies | contradicts | replicates | aggregates | part_of_model | background | apparent_tension",
  "incompleteness_category": "effect_disappears_under_control | hub_component | boundary_condition | extends_prior_definition | none",
  "confidence_score": 0.85
}
```

Framework: components → whole via `part_of_model` + `hub_component`.
Motivation → proposal: `background`.

### paper_references

Substantively used references only (not parenthetical lists). Typical 10-25.
```json
{"cited_doi": "10.xxxx/yyyy", "cited_title": "Title"}
```
Do not fabricate DOIs.

## Reading order

1. Title + Abstract → paper metadata, title_concepts, motivation EXISTENCE_CLAIMs.
2. Method → `default_method`, `default_causal_strength`. Judge by **research design**, not word choice.
3. Results + Tables + Figures → each Table maps to ≥1 claim. Record all baselines.
4. Discussion → usually `verification_status = proposed_in_paper`. Override causal_strength only when different.
5. References → 10-25 substantively used.

## iv/dv naming

`{unit_of_analysis}_{concept}`. Based on **Method measurements**, not Discussion abstractions. If Discussion uses a broader label → `author_generalization`. Measurement differences (self-report vs observer) → `measurement_methods`, not iv/dv.

## causal_strength

Benchmark / ablation → `author_framing: existence_proof`, `temporal_precedence: none`.
"outperforms", "SOTA" → `author_framing: existence_proof`.
"may", "could", "will be" → `author_framing: suggestive`.
Cross-sectional with causal wording → `design_basis: cross_sectional, author_framing: causal` (record the gap).

## Appendix A: evidence_type

```
meta_analysis, rct, quasi_experimental, longitudinal_field,
cross_sectional_field, qualitative_field, computational_demonstration,
theoretical_derivation, theoretical_integration, re_interpretation,
expert_framework, conceptual_argument
```

## Appendix B: l3_json fields

Record only applicable fields.

Stats: `effect_size, effect_size_type, p, ci_lower, ci_upper, n, r_squared, r_squared_type, k, N_total, heterogeneity_I2, publication_bias`.

Structured:
```
mediation: {total_effect, direct_effect, indirect_effect, direct_p, indirect_p, proportion_mediated, note}
interaction: {variable: value}
models: [{label, effect_size, effect_size_type, ...}]    ← all baselines
reliability: {alpha, icc, ...}
formal: {equation, variables, conditions, proof_type}
additional: {...}                                         ← required for conceptual papers
```

`effect_size_type`:
- Psychology: `B, beta, r, rho, d, eta_squared, partial_eta_squared, omega_squared, odds_ratio, hazard_ratio`
- ML: `BLEU, accuracy, F1, perplexity, Exact_Match, ROUGE, AUC, MAE, MSE`

## Appendix C: incompleteness_category

```
effect_disappears_under_control — effect vanishes when controlling
hub_component                   — component of mediation/moderation model
boundary_condition              — boundary condition stated
extends_prior_definition        — extends a prior definition
none                            — standalone citation safe
```

## Appendix D: concept prefixes

`team_`, `individual_`, `organizational_`, `computational_`, `gene_`, `national_`, `dyadic_`, `channel_`, `source_`.

OB team-level consistent names: `team_psychological_safety`, `team_learning_behavior`, `team_performance`, `team_context_support`, `team_leader_coaching`, `team_efficacy`, `team_type`.
