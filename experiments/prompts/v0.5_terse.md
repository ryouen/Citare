# Citare Extract v0.5 (terse)

Temperature = 0. Extract every substantive claim from this paper as a single JSON object.

**Goal**: structured claims usable as a machine-readable database. Each claim must prevent two errors:
1. **Causal misattribution** — writing "causes" when data shows "associated with"
2. **Missing key findings** — when unsure, output with lower confidence_score rather than omit

## Output: single JSON object

```json
{"paper": {...}, "claims": [...], "theories": [...], "measurement_methods": [...], "claim_relations": [...], "paper_references": [...]}
```

### paper

```json
{
  "doi": "10.xxxx/yyyy",
  "title": "...",
  "title_concepts": ["{unit}_{concept}", ...],
  "authors": ["..."],
  "year": 2024,
  "venue": "...",
  "paper_type": "empirical | conceptual | review | meta_analysis | book | book_chapter",
  "domain": "e.g. organizational_behavior, ml, ai_safety",
  "default_method": {"sample_size": N, "unit_of_analysis": "team|individual|organization|computational|gene|national|dyadic", "industry": "...", "country": "...", "study_design": "..."},
  "default_causal_strength": {
    "design_basis": "cross_sectional | longitudinal | quasi_experimental | rct | meta_analysis | computational_demonstration | theoretical",
    "temporal_precedence": "none | partial | full",
    "author_framing": "causal | associational | suggestive | existence_proof",
    "manipulation_of_iv": true/false
  }
}
```

default_method / default_causal_strength describe the paper's primary design. Individual claims override these **only** when they differ.

### claims (4 templates)

Every claim:
```json
{
  "id": "{first_author_surname_lc}{year}_{def|rel|exist|meta}{n}",
  "template_type": "DEFINITION | RELATION | EXISTENCE_CLAIM | META_CLAIM",
  "l0_json": {...},
  "l3_json": {...},        // statistics; omit for non-numerical claims
  "source_text": "1-3 sentence direct quote",
  "source_page": <printed page number>,
  "source_section": "...",
  "evidence_type": "see Appendix A",
  "verification_status": "verified_in_paper | proposed_in_paper | null",
  "confidence_score": 0.0-1.0,
  "causal_strength": {...}, // override only when different from paper default
  "method_metadata": {...}   // override only when different from paper default
}
```

**No l1 or l2 fields.** Clients generate natural-language + triples from l0_json + source_text at read time.

Minimum counts: empirical ≥ 5 claims; conceptual ≥ 8.

#### DEFINITION — "what X is"
```json
"l0_json": {"concept": "{unit}_{concept}", "key_elements": [...], "distinguished_from": [...]}
```
verification_status = null. For framework papers: **separate DEFINITION for each component**, not one for the whole.

#### RELATION — "X is related to Y"
```json
"l0_json": {
  "iv": "{unit}_{concept}",
  "dv": "{unit}_{concept}",
  "relation": "positive | negative | null | curvilinear | conditional",
  "mediator": null,
  "moderator": null,
  "author_generalization": null
}
```
iv/dv are based on **Method section measurements**, not Discussion abstractions. When Discussion uses a more abstract label than what was measured:
```json
"author_generalization": {"discussion_label": "wellbeing", "actual_measure": "individual_life_satisfaction_swls"}
```
For hypotheses: verification_status per Results. Baseline comparison / ablation → author_framing = "existence_proof".

#### EXISTENCE_CLAIM — "X was observed / has a limitation / was demonstrated"
```json
"l0_json": {"phenomenon": "...", "evidence": "1-2 sentence evidence"}
```
Use for: study design limitations, paper motivation, capability demonstration, boundary conditions, non-relational numerical findings (e.g., ICC=.39).

#### META_CLAIM — "integrative finding across studies or the paper"
```json
"l0_json": {"integrated_finding": "...", "synthesis_type": "quantitative_meta_analysis | qualitative_synthesis | theoretical_integration", "k": null, "N_total": null}
```

### theories
Record only **explicitly named** theories (organizes 3+ claims, has mechanism, generates hypotheses):
```json
[{"name": "...", "type": "theory | theoretical_model | framework", "component_claims": [...], "description": "...", "scope": "..."}]
```

### measurement_methods
```json
[{"id": "short_name", "name": "...", "subtype": "questionnaire_scale | behavioral_observation | computational_benchmark | physiological_measure | experimental_paradigm | interview | content_analysis | archival_data", "measures": "concept_name", "details": {"item_count": N, "response_scale": "...", "reverse_items": [1,3,5], "reliability_reported": 0.82, "original_source": "...", "copyright_status": "public | proprietary"}, "used_in_claims": [...]}]
```

`reverse_items`: 1-based item numbers, not item text. `reliability_reported`: numeric (0.82), not string.

### claim_relations
Record every relationship you find. Use confidence_score for uncertainty, don't omit.

```json
[{"source_id": "...", "target_id": "...", "relation_type": "supports | extends | qualifies | contradicts | replicates | aggregates | part_of_model | background | apparent_tension", "incompleteness_category": "effect_disappears_under_control | hub_component | boundary_condition | extends_prior_definition | none", "confidence_score": 0.85}]
```

Framework papers: components → whole via `part_of_model` + `hub_component`. Motivation → proposal: `background`.

### paper_references
Record only references substantively used (findings, definitions, or frameworks built upon; results compared or extended; cited with specific findings). Skip parenthetical bulk lists. Typical: 10-25.

```json
[{"cited_doi": "10.xxxx/yyyy", "cited_title": "..."}]
```
Fabricate no DOIs. Use null if uncertain.

## Reading order

1. **Title + Abstract** → paper metadata + title_concepts + motivation EXISTENCE_CLAIMs
2. **Method** → default_method, default_causal_strength. Judge by **design**, not the author's word choice
3. **Results + Tables + Figures** → each Table → ≥1 RELATION or EXISTENCE_CLAIM. Read numbers directly. Record **all baselines**, not just the winner
4. **Discussion** → usually verification_status = "proposed_in_paper". Override causal_strength only when different
5. **References** → 10-25 substantively used

## Judgment

**verification_status**: data-backed → verified; untested interpretation → proposed; mixed → split into two claims; DEFINITION → null.

**causal_strength** (applied at paper.default_causal_strength):
- benchmark / ablation → temporal_precedence=none, author_framing=existence_proof
- "outperforms", "achieves SOTA" → author_framing=existence_proof
- "may", "could", "will be" → author_framing=suggestive

**iv/dv naming**: `{unit_of_analysis}_{concept}`. Based on Method measurements. Method-Discussion gap → author_generalization. Self-report vs observer-rated → measurement_methods, not iv/dv.

## Appendix A: evidence_type

```
meta_analysis, rct, quasi_experimental, longitudinal_field, cross_sectional_field, qualitative_field, computational_demonstration, theoretical_derivation, theoretical_integration, re_interpretation, expert_framework, conceptual_argument
```

## Appendix B: l3_json fields

Record only applicable fields.

**Statistics**: effect_size, effect_size_type, p, ci_lower, ci_upper, n, r_squared, r_squared_type, k, N_total, heterogeneity_I2, publication_bias

**Structured**:
- mediation: {total_effect, direct_effect, indirect_effect, direct_p, indirect_p, proportion_mediated, note}
- interaction: {variable: value}
- models: [{label, effect_size, effect_size_type, ...}] ← **all baselines**
- reliability: {alpha, icc, ...}
- formal: {equation, variables, conditions, proof_type}
- additional: {any structural info} ← required for conceptual papers

**effect_size_type** (psychology): B, beta, r, rho, d, eta_squared, partial_eta_squared, omega_squared, odds_ratio, hazard_ratio
**effect_size_type** (ML): BLEU, accuracy, F1, perplexity, Exact_Match, ROUGE, AUC, MAE, MSE

## Appendix C: incompleteness_category

```
effect_disappears_under_control — effect vanishes when controlling
hub_component                   — component of mediation/moderation model
boundary_condition              — boundary condition exists
extends_prior_definition        — extends a prior definition
none                            — can be cited standalone
```

## Appendix D: Concept prefix

Use `{unit_of_analysis}_{concept}`. Examples: `team_`, `individual_`, `organizational_`, `computational_`, `gene_`, `national_`, `dyadic_`, `channel_`, `source_`.

Consistent concepts (prefer these names when applicable):
- OB team-level: team_psychological_safety, team_learning_behavior, team_performance, team_context_support, team_leader_coaching, team_efficacy, team_type
